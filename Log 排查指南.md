# grep 常用命令及正则实战指南

`grep` (Global Regular Expression Print) 是 Linux 下最强大的文本搜索工具。它的核心逻辑是：**输入文本 -> 模式匹配 -> 输出行**。

## 1. grep 常用参数 (Options)

|**参数**|**说明**|**示例**|
|---|---|---|
|**`-i`**|**忽略大小写** (Ignore case)|`grep -i "error" log.txt`|
|**`-v`**|**反向过滤** (Invert)|`grep -v "DEBUG" log.txt` (不看DEBUG日志)|
|**`-n`**|**显示行号** (Line number)|`grep -n "25405" log.txt`|
|**`-w`**|**精确匹配单词** (Word)|`grep -w "dev" log.txt` (匹配 dev，不匹配 device)|
|**`-E`**|**开启扩展正则** (Extended)|`grep -E "A\|B"` (匹配 A 或 B)|
|**`-r`**|**递归搜索目录** (Recursive)|`grep -r "25405" /var/logs/`|
|**`-o`**|**只输出匹配到的部分**|`grep -o "topic: [^,]*"` (只提取 topic 内容)|
|**`-l`**|**只列出文件名**|`grep -l "ERROR" *.log` (找出哪些文件有报错)|

## 2. 常用正则表达式 (Regex)

在使用复杂逻辑时，建议配合 `-E` 参数使用。

### 1. 基础匹配
- **`^` (行首)**：`grep "^2026"` (匹配以 2026 开头的行)
- **`$` (行尾)**：`grep "success$"` (匹配以 success 结尾的行)
- **`.` (任意单个字符)**：`grep "d.v"` (匹配 dev, dav, d1v 等)

### 2. 数量与重复
- **`*` (零个或多个)**：`grep "abc*"` (匹配 ab, abc, abcc...)
- **`+` (一个或多个)**：`grep -E "abc+"` (至少有一个 c)
- **`?` (零个或一个)**：`grep -E "https?"` (匹配 http 或 https)

### 3. 分组与分支 (关键)
- **`|` (或)**：`grep -E "ERROR|WARN|CRITICAL"`
- **`[]` (字符集)**：`grep "device[1-3]"` (匹配 device1, device2, device3)
- **`[^]` (排除集)**：`grep "[^0-9]"` (匹配非数字字符)

## 将查询结果输出到文件

在调试时，我们通常将过滤后的“精华”内容保存下来，方便后续分析或发送给同事。

### 1. 覆盖写入 (`>`)

将结果保存到新文件。如果文件已存在，会**直接覆盖**原内容。
```bash
grep "25405" star-anti-uav-debug.log > device_25405_analysis.log
```
### 2. 追加写入 (`>>`)

将结果添加到已有文件的末尾，**不会删除**原内容。
```bash
grep "ERROR" star-anti-uav-debug.log >> all_errors.txt
```

### 3. 同时输出到屏幕和文件 (`tee`)

有时候你想边看屏幕输出，边保存到文件，可以使用 `tee` 命令。

```bash
grep "DEVICE_INFO" star-anti-uav-debug.log | tee search_result.log
```

## 综合案例实战
### 搜索多个可能的关键字（或关系）

找出所有包含 "fail"、"error" 或 "exception" 的日志，忽略大小写：


```bash
grep -Ei "fail|error|exception" star-anti-uav-debug.log > fatal_issues.log
```

### 复杂的连续过滤（与关系）

找出 2026-03-12 这一天，设备 25405 的所有 MQTT 成功消息：

```bash
grep "2026-03-12" star-anti-uav-debug.log | grep "25405" | grep "MQTT消息成功" > success_flow.log
```

# 普通日志查询方案

### 1. 精确范围查询 (核心案例)

**场景**：查找 08:09:45 到 09:09:45 之间，关于设备 `25405` 的所有 `DEVICE_INFO` 日志。

```bash
awk '$1 == "2026-03-12" && $2 >= "08:09:45.828" && $2 <= "09:09:45.828"' star-anti-uav-debug.log \
| grep "DEVICE_INFO" \
| grep "25405"
```

- **说明**：`awk` 负责毫秒级的时间截取，两个 `grep` 确保结果同时包含两个关键字。

### 2. 故障频率统计

**场景**：统计 10 点到 11 点之间，MQTT 发送失败的总次数。

```bash
awk '$2 >= "10:00:00" && $2 <= "11:00:00"' star-anti-uav-debug.log \
| grep "发送MQTT消息" \
| grep "失败" \
| wc -l
```

- **说明**：`wc -l` 用于统计行数，快速评估故障严重程度。

### 3. 排查上下文 (前后关联)

**场景**：发现 09:05:01 有个报错，想看这个报错前后各 5 行的逻辑（如 MQTT 报文细节）。

```bash
awk '$2 >= "09:04:50" && $2 <= "09:05:10"' star-anti-uav-debug.log \
| grep -C 5 "ERROR"
```

- **说明**：`-C 5` (Context) 会显示匹配行及其上下各 5 行，非常适合分析异步调用。

### 4. 排除干扰信息 (反向过滤)

**场景**：查看 08:00 后的设备日志，但排除掉刷屏的“心跳包（HEARTBEAT）”。

```bash
awk '$2 >= "08:00:00"' star-anti-uav-debug.log \
| grep "25405" \
| grep -v "HEARTBEAT"
```

- **说明**：`grep -v` 排除不需要的干扰项，让关键日志更显眼。

### 5. 实时监控特定范围

**场景**：实时查看 10 点以后新产生的设备 `25405` 的调试信息。

```bash
tail -f star-anti-uav-debug.log \
| awk '$2 >= "10:00:00"' \
| grep --line-buffered "25405"
```

- **说明**：`--line-buffered` 解决管道缓存导致的显示延迟。

### 6. 提取日志中的特定字段

假设你想看看 08:00 到 09:00 之间，设备 25405 发送的所有 Topic 是什么。

```bash
awk '$2 >= "08:00:00" && $2 <= "09:00:00"' star-anti-uav-debug.log | grep "25405" | awk -F "topic: " '{print $2}' | awk -F "," '{print $1}'
```
**说明**：
- `-F "topic: "`：以字符串 `topic:` 作为分隔符，取出后面的部分。
- 通过多次管道，你可以把杂乱的日志变成整齐的列表。
- 
## 工具参数速查表

|**工具**|**参数**|**含义**|**用途举例**|
|---|---|---|---|
|**awk**|`$1, $2`|第一列、第二列|对应日志中的 `日期` 和 `时间`|
|**awk**|`&&`|逻辑与|同时满足日期和时间范围条件|
|**grep**|`-E`|扩展正则|`grep -E "A\|B"` (匹配 A 或 B)|
|**grep**|`-i`|忽略大小写|`grep -i "error"`|
|**grep**|`-v`|反向选择|`grep -v "DEBUG"` (只看非 DEBUG 日志)|
|**grep**|`-C n`|上下文|`grep -C 3` (看前后各 3 行)|
|**wc**|`-l`|统计行数|算出某段时间报错了多少次|

# 压缩日志 (.gz) 批量查询方案

## 1. 核心工具介绍

- **`zgrep`**：最常用的工具，直接在压缩包内搜索，无需手动解压。
    
- **`zcat`**：将压缩包内容流式输出到屏幕（类似 `cat`），常配合 `awk` 使用。
    
- **`zless`**：像 `less` 一样查看压缩包内容，支持翻页和 `/` 搜索。
    

---

## 2. 批量查询案例

### A. 在多个压缩包中搜索关键字

**场景**：在所有的历史日志中查找设备 `25405` 的信息。

```
zgrep "25405" star-anti-uav-debug.log.*.gz
```

- **技巧**：如果想知道匹配项具体在哪个压缩包里，`zgrep` 默认会显示文件名。

### B. 结合 awk 进行时间段拦截 (黄金组合 2.0)

**场景**：从压缩包中提取 2026-03-12 08:00 到 09:00 之间的日志，并过滤关键字。

```bash
zcat star-anti-uav-debug.log.*.gz | awk '$2 >= "08:00:00" && $2 <= "09:00:00"' | grep "25405"
```

- **说明**：`zcat` 会把所有匹配的文件按顺序解压成流，交给 `awk` 处理。
    

### C. 跨越“压缩”与“未压缩”日志 (全量查询)

**场景**：你不知道日志是在当前的 `.log` 里，还是在历史的 `.log.gz` 里。

```bash
# 使用 zgrep -f 配合通配符，zgrep 通常也能处理未压缩的普通文本
zgrep "DEVICE_INFO.*25405" star-anti-uav-debug.log*
```

---

## 3. 高级进阶：按日期批量查询

通常日志轮转后文件名会带日期，比如 `debug.log.2026-03-11.gz`。

**场景**：只查询 3 月 10 号到 3 月 12 号这三天的压缩日志。

```bash
# 使用大括号扩展文件名
zgrep "25405" star-anti-uav-debug.log.2026-03-{10,11,12}.gz
```

---

## 4. 性能优化方案

如果日志文件非常多（例如有上百个 `.gz`），单线程搜索会很慢。可以使用 `xargs` 开启多进程并行搜索：

```bash
# 开启 4 个进程并行搜索，速度大幅提升
ls star-anti-uav-debug.log.*.gz | xargs -P 4 -I {} zgrep "25405" {}
```

## 压缩包查询命令对比表

|**命令**|**适用场景**|**备注**|
|---|---|---|
|**`zgrep "key" *.gz`**|快速查找关键字|最常用，省去解压步骤|
|**`zcat *.gz \| awk ...`**|需要复杂逻辑过滤|适合处理带时间戳的范围过滤|
|**`zgrep -c "key" *.gz`**|统计每个压缩包中的出现次数|快速判断哪天的数据有问题|
|**`zgrep -l "key" *.gz`**|只看哪些文件包含关键字|不显示具体内容，只看文件名|
